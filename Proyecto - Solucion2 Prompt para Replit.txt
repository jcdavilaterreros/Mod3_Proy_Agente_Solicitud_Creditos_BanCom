La app debe ser 100% funcional en el navegador y NO debe depender de backend. Debe ser mobile-first y con máximo ancho 448px (layout tipo WhatsApp). Sin errores en consola. Código claro, modular y comentado.

=================================
OBJETIVO GENERAL DE LA APLICACIÓN
=================================
Crear un prototipo tipo WhatsApp, integrado a una landing simple, para concretar una solicitud de préstamo. Debe sentirse conversación casual (no interrogatorio). Las respuestas del agente deben sonar naturales, cálidas, claras y accionables. La app debe validar que el usuario reciba respuesta por cada mensaje y también validar que se hayan “cargado” los documentos requeridos antes de pasar a evaluación.

Aunque se menciona GPT-4o-mini, como este MVP es sin backend:
- Implementa un motor “LLM Simulator” (respuestas humanas simuladas) que use el estado del flujo para responder.
- Deja preparada una función callLLM(promptPayload) para futura integración con GPT-4o-mini, pero desactivada por defecto (no llamar APIs reales).
- Incluye un toggle “Modo IA (simulado / integrado)” que por defecto sea “Simulado” y el modo “Integrado” muestre un mensaje: “Pendiente de backend / proxy seguro”.

- Subido (verde #22c55e)
- Observado (ámbar #f59e0b)

Interacción:
- Botón “Subir” por cada documento que simula carga (sin archivos reales). Al click:

  - Un botón contextual “Marcar observado” (solo visible en modo debug) para probar UX.
  - Si un doc está “Observado”, el agente debe pedir reintento con tono cordial.
- La app no permite pasar a evaluación si algún documento está Pendiente u Observado.

Texto exacto cuando se entra a documentos:
"¡Perfecto! Para continuar con tu solicitud, necesito que subas los siguientes documentos:"

- documentos recibidos

- Si monto <= 15000 y plazo <= 24: “Aprobada”
